**Free addon: New Collection with Selection — one click instead of dragging objects into an empty collection**

Blender hides a handy operator in the M menu: **Move to Collection ▸ New Collection**
creates a collection and moves your selection in. But that's not where your hand goes. You
select a few objects, reach for the Outliner's **New Collection** button — and it ignores
your selection, handing you an empty collection with your objects still loose. So you drag
them in one by one, or go digging for the M-menu entry.

This addon adds a **New Collection with Selection** command to the Outliner right-click
menu and the viewport's Object > Collection menu — the same create-and-move you'd get from
the M menu, surfaced where you actually click. One click: a new collection appears, nested
under the active collection, with your selected objects already inside it. It's a true
move (no duplicate links) and fully undoable.

## Long (Newsletter / Blog post intro)

**The empty-collection problem, fixed**

Here's a workflow many Blender users encounter. You've got a handful of objects you want to
group — a character's props, a set of modules, a few lights. Blender can do this already:
**M ▸ Move to Collection ▸ New Collection** creates a fresh collection and moves your
selection in. But that's a menu you have to remember to open. What you actually reach for
is the **New Collection** button in the Outliner — and that one creates an empty
collection and leaves your objects exactly where they were. So you drag them in by hand,
or undo and go hunting for the New Collection entry buried in the M menu. People have been
asking for the Outliner button to respect the selection since 2020.

**New Collection with Selection** closes the gap — not by inventing anything, but by
putting the create-and-move you already have where you'd expect to find it. It
adds a command of the same name to the two places you create collections — the Outliner
right-click menu and the viewport's Object > Collection menu. Select your objects, run it,
and you get a new collection nested under the active one with your selection already
inside, set as the active collection so your next addition lands there too. Same nesting,
same true-move semantics as the M-menu route (each object ends up in one collection, no
duplicates), fully undoable.

No preferences to configure, no background behavior to second-guess — install it, enable
it, and the command is waiting in the menus you already use.
