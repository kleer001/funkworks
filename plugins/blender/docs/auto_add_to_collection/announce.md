This addon adds a **New Collection with Selection** command to the Outliner right-click

## Long (Newsletter / Blog post intro)

**The empty-collection paper cut, fixed**

Here's a workflow many Blender users encounter. You've got a handful of objects you want to
group — a character's props, a set of modules, a few lights. You select them, click the
New Collection button in the Outliner, and... Blender creates an empty collection and
leaves your objects exactly where they were. So you drag them in by hand, or undo and
reach for the M (Move to Collection) menu, which lives somewhere else entirely. The
Outliner's New Collection action just doesn't look at your selection. People have been
asking for this to change since 2020.

**New Collection with Selection** closes the gap. It adds a command of the same name to
the two places you already create collections — the Outliner right-click menu and the
viewport's Object > Collection menu. Select your objects, run it, and you get a new
collection nested under the active one with your selection already inside, set as the
active collection so your next addition lands there too. It behaves exactly like the
native New Collection button otherwise: same nesting, same true-move semantics (each
object ends up in one collection, no duplicates), fully undoable.

No preferences to configure, no background behavior to second-guess — install it, enable
it, and the command is waiting in the menus you already use.
